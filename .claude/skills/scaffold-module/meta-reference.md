# meta.json Reference

## Template

```json
{
  "label": "Module Name",
  "description": "Brief description of what the module does",
  "icon": "../../base/modules/icons/icon-name.svg",
  "is_available_for_new_content": true,
  "host_template_types": ["PAGE", "BLOG_POST", "BLOG_LISTING"],
  "categories": ["BODY_CONTENT"],
  "smart_type": "NOT_SMART"
}
```

---

## Rules

### Required fields

- `label` — Title Case with spaces (e.g. `"Feature Cards"`)
- `description` — always present; missing description can trigger `"internal error"`
- `icon` — relative SVG path: `"../../base/modules/icons/name.svg"`
- `is_available_for_new_content` — always `true`
- `host_template_types` — MANDATORY when CSS or JS files exist; use `["PAGE", "BLOG_POST", "BLOG_LISTING"]`
- `categories` — at least one UPPERCASE category
- `smart_type` — `"NOT_SMART"`

### Valid categories (UPPERCASE only)

- `TEXT` — Text-based modules
- `COMMERCE` — E-commerce/shopping modules
- `DESIGN` — Layout/design modules
- `FUNCTIONALITY` — Interactive functionality modules
- `FORMS_AND_BUTTONS` — Forms, buttons, CTAs
- `BODY_CONTENT` — Main content modules (accordions, content blocks, etc.)
- `MEDIA` — Image/video/media modules
- `BLOG` — Blog-specific modules
- `SOCIAL` — Social media modules

**NEVER** use lowercase like `"content"` — it will fail validation.

### Forbidden fields (auto-generated by HubSpot)

Do NOT include any of these:

- `content_tags` — causes `Cannot deserialize value of type java.util.ArrayList` error
- `module_id` — auto-generated on upload
- `global` — not used in Core Code
- `content_types` — redundant with `host_template_types`
- `css_assets` / `js_assets` — assets are loaded in the template via `require_css`/`require_js`

### Label formatting

Convert module directory name to Title Case with spaces:

| Directory | Label |
|---|---|
| `image_carousel.module` | `"Image Carousel"` |
| `feature_cards.module` | `"Feature Cards"` |
| `call_to_action.module` | `"Call To Action"` |

### Icon format

Use relative SVG path: `"../../base/modules/icons/name.svg"`

Alternative FontAwesome format: `"fontawesome-5.14.0:IconName:SOLID"` or `"fontawesome-5.14.0:IconName:REGULAR"`

## HubSpot upload error reference

For the complete upload error table with root causes and fixes, see `.claude/skills/fix-module/SKILL.md`. Use `/fix-module` to diagnose and fix errors automatically.

**Key meta.json rules that prevent upload errors:**
- Always include `description` and `icon` (missing either triggers `internal error`)
- Always include `host_template_types` when `css_assets` or `js_assets` are present
- Categories MUST be UPPERCASE (`BODY_CONTENT`, not `body_content`)
- Never include `content_tags` or `module_id` (auto-generated by HubSpot)
